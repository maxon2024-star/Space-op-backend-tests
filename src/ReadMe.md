# Space Operations API - Автоматизированное тестирование

Этот репозиторий содержит проект автоматизированного тестирования для основного REST API сервиса "Space Operations Center". Проект реализован в рамках курса по системному дизайну и реализует подход Black-Box тестирования.

## 🛠 Стек технологий

* **Язык:** Java 21
* **Сборщик:** Gradle (Kotlin DSL)
* **Фреймворк для тестирования:** JUnit 5 (Jupiter)
* **API Клиент:** REST Assured
* **Отчетность:** Allure Report
* **Сериализация:** Jackson Databind

## 📋 Предварительные требования

Перед запуском автотестов убедитесь, что соблюдены следующие условия:

1.  Установлен **JDK 17** (или выше).
2.  Основной сервис **Space Operation Center** запущен локально и слушает порт `7070` (доступен по адресу `http://localhost:7070`).
3.  База данных основного проекта (PostgreSQL) поднята и доступна.

##  Запуск автотестов

Проект использует Gradle Wrapper, поэтому вам не нужно устанавливать Gradle глобально в систему.

Откройте терминал в корневой папке данного репозитория и выполните следующую команду:

**Для Windows:**
```bash
gradlew clean test

```

**Для Linux / macOS:**

```bash
./gradlew clean test

```

Эта команда скачает все необходимые зависимости, скомпилирует тестовые классы и последовательно выполнит сценарии тестирования.

##  Генерация и просмотр отчета Allure

Проект настроен на автоматическое логирование всех HTTP-запросов и ответов в формате, понятном для Allure.

Чтобы сгенерировать красивый HTML-отчет и открыть его в браузере, выполните команду:

**Для Windows:**

```bash
gradlew allureServe

```

**Для Linux / macOS:**

```bash
./gradlew allureServe

```

Локальный сервер Allure поднимется автоматически и откроет вкладку в вашем браузере по умолчанию. Чтобы остановить сервер Allure, нажмите `Ctrl + C` в терминале.

##  Структура тестовых сценариев

Тесты запускаются в строгом порядке, имитируя полный жизненный цикл работы с группировками:

1. `testCreateConstellation` — Успешное создание новой спутниковой группировки.
2. `testAddSatellite` — Успешное добавление спутника связи в созданную группировку.
3. `testExecuteMissionForSingleSatellite` — Успешный запуск миссии для одиночного спутника.
4. `testGetOverview` — Получение общей сводки системы и проверка наличия наших данных.
5. `testDecommissionSatellite` — Вывод спутника из эксплуатации (очистка данных).

*Примечание: Имена группировок и спутников генерируются динамически с использованием UUID для предотвращения конфликтов уникальности в базе данных (State Leakage) при повторных запусках.*
